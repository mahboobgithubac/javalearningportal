What is the Need for Method Reference in Java?

In Java 8, method references were introduced to simplify lambda expressions that only call existing methods. They make code cleaner, more readable, and less verbose.

1. Avoid Rewriting Logic Already in Methods

// Lambda expression
list.forEach(item -> System.out.println(item));

// Method reference
list.forEach(System.out::println);

Why? Because the logic already exists ‚Äî no need to rewrite it.

2. Improves Readability

// Using lambda
Function<String, Integer> strToInt = str -> Integer.parseInt(str);

// Using method reference
Function<String, Integer> strToInt = Integer::parseInt;

3. Eliminates Boilerplate Code

// Lambda
BiFunction<Integer, Integer, Integer> sum = (a, b) -> Math.max(a, b);

// Method reference
BiFunction<Integer, Integer, Integer> sum = Math::max;

4. Used Heavily in Functional Interfaces

List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

// Using lambda
names.forEach(name -> System.out.println(name));

// Using method reference
names.forEach(System.out::println);

5. Used in Stream Pipelines

List<String> list = Arrays.asList("1", "2", "3");

// Using method reference instead of str -> Integer.parseInt(str)
List<Integer> intList = list.stream()
    .map(Integer::parseInt)
    .collect(Collectors.toList());
