Git Update Guide

1. Initial Git Setup

Use these commands in your project root to initialize and push code to GitHub:

git init
git remote add origin https://github.com/username/repo-name.git
git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main

2. Pushing Updates to GitHub

After modifying files, use the following to push updates:

git add .
git commit -m "Updated some files"
git push origin main

Note: Ensure you are on the correct branch (e.g., main).

3. Check Remote Repository

To verify or change your remote URL:

git remote -v

To change remote URL:

git remote set-url origin https://github.com/username/new-repo.git

4. Fix: Push is Disabled

If STS or terminal says "push is disabled", ensure:

  • Youâ€™ve added a remote: git remote -v
  • You're on the correct branch: git branch
  • Your current branch is tracking the remote branch: git branch -u origin/main

Or use this if not set already:

git push --set-upstream origin main

5. View Commit History

git log --oneline

6. STS Git Integration (Optional)

In Spring Tool Suite (STS), go to:

  • Right-click project → Team → Commit
  • Enter commit message
  • Click Push or Push to Upstream
