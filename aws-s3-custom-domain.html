AWS S3 File Upload with Custom Domain

When uploading files to Amazon S3, the default URL format usually looks like:

https://my-bucket.s3.amazonaws.com/myfile.jpg

Can we replace .s3.amazonaws.com with a custom domain?

Yes! You can set up a custom domain so that your file URL looks like:

https://cdn.mywebsite.com/myfile.jpg
This requires some configuration in AWS S3, CloudFront, and your domain DNS.

Steps to Use a Custom Domain for S3 Files

  1. Create or use an existing S3 bucket and make sure it is public for file access or has signed URL configuration.
  2. Set up an Amazon CloudFront distribution pointing to your S3 bucket as the origin.
  3. Configure your custom domain in CloudFront:
    • Add an Alternate Domain Name (CNAME) such as cdn.mywebsite.com.
    • Attach an SSL/TLS certificate from AWS Certificate Manager (ACM).
  4. Update your DNS records in Route 53 or your DNS provider to point cdn.mywebsite.com to the CloudFront distribution.
  5. Use the custom domain in your application instead of the default .s3.amazonaws.com URL.

Advantages of Using a Custom Domain

  • Branding consistency with your own domain.
  • Better SEO for images and files.
  • Improved performance via CloudFront's global CDN.
  • Easier to migrate files without changing app code (only update DNS).
