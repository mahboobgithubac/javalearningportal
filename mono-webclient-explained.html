Mono<String> means: "You'll get a String later, asynchronously."

‚úÖ What is Mono?

  • Mono is a reactive type that represents 0 or 1 result.
  • It is part of Project Reactor used in Spring WebFlux.
  • It is non-blocking and async‚Äîgreat for high-performance applications.

üìå Think of Mono Like...

Concept Description
Mono<T> Will emit 0 or 1 item of type T (e.g., Mono<String>)
Promise / Future Similar idea but Mono is reactive
Optional + Async One item, maybe empty, and delivered in future

üöÄ How to Use Mono

Option 1: Blocking (not preferred)

String result = response.block();

Waits until the response is ready and gives the value.

Option 2: Subscribing (reactive style)

response.subscribe(data -> System.out.println("Received: " + data));

Registers a callback to be called when the data is available.
