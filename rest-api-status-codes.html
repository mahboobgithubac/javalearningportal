REST API HTTP Status Codes

Your quick reference for web development and backend APIs

1xx – Informational

CodeMeaningDescription
100ContinueThe server has received the request headers and the client can proceed to send the request body.
101Switching ProtocolsProtocol upgrade initiated by the client is accepted by the server.

2xx – Success

CodeMeaningDescription
200OKRequest succeeded. Typically returned for GET or PUT.
201CreatedResource successfully created. Used with POST.
202AcceptedRequest accepted for processing, but not yet completed.
204No ContentRequest succeeded but no content returned (used with DELETE, PUT).

3xx – Redirection

CodeMeaningDescription
301Moved PermanentlyResource permanently moved to a new URL.
302FoundTemporarily redirected.
304Not ModifiedClient can use cached version.

4xx – Client Errors

CodeMeaningDescription
400Bad RequestMalformed request syntax or validation error.
401UnauthorizedAuthentication required or failed.
403ForbiddenAuthenticated but not allowed to access the resource.
404Not FoundResource doesn't exist.
405Method Not AllowedHTTP method is not supported for the resource.
409ConflictRequest conflicts with the current state of the resource.
422Unprocessable EntityRequest is well-formed but semantic errors occurred.

5xx – Server Errors

CodeMeaningDescription
500Internal Server ErrorUnexpected server error.
501Not ImplementedThe server doesn't support the requested functionality.
502Bad GatewayInvalid response from the upstream server.
503Service UnavailableThe server is down or overloaded.
504Gateway TimeoutThe server didn’t receive a timely response from upstream.
