What is a String in Java?

A String is a sequence of characters. It is immutable and used to represent text.

Ways to Create Strings

// Using string literal
String str1 = "Hello";

// Using new keyword
String str2 = new String("World");

Common String Methods

public class StringExample {
    public static void main(String[] args) {
        String s = "Java Programming";

        System.out.println("Length: " + s.length());
        System.out.println("Uppercase: " + s.toUpperCase());
        System.out.println("Lowercase: " + s.toLowerCase());
        System.out.println("Char at index 5: " + s.charAt(5));
        System.out.println("Substring: " + s.substring(5, 16));
        System.out.println("Contains 'Java'? " + s.contains("Java"));
        System.out.println("Replace 'Java' with 'C++': " + s.replace("Java", "C++"));
    }
}

Practice Questions

Q1: What is the difference between == and .equals()?

Answer: == compares memory reference, while .equals() compares content.

String a = "hello";
String b = new String("hello");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
Q2: Is String mutable in Java?

Answer: No, it is immutable. Every modification creates a new object.

ðŸ”’ Why is String Immutable in Java?

In Java, the String class is immutable, which means once a String object is created, it cannot be changed. Any operation that seems to modify a string actually creates a new String object.

âœ… Reasons Why String is Immutable:

1. Security

Strings are used in network connections, file paths, class loading, etc. If strings were mutable, malicious code could alter them after creation.

String path = "/usr/bin";
// If path could be changed maliciously

2. Caching in String Pool

Java uses a String Pool to reuse objects. Two strings with the same content can refer to the same memory location. Immutability ensures consistency of these shared strings.

String s1 = "hello";
String s2 = "hello";
System.out.println(s1 == s2); // true (same object from pool)

3. Thread Safety

Immutable objects are inherently thread-safe. No need for synchronization when strings are shared between threads.

4. Hashcode Caching

Strings are often used as keys in HashMaps. Immutability ensures hashCode doesn't change, so key lookup remains consistent.

Map<String, String> map = new HashMap<>();
map.put("user", "Mahboob");

// If the key changes, you canâ€™t retrieve it

5. Predictable Behavior

Immutable objects behave more predictably and are easier to debug.

String vs StringBuffer vs StringBuilder in Java

1. String

Immutable: Every modification creates a new object. 

String str = "Hello";
str.concat(" World"); // Does not change the original string
System.out.println(str); // Output: Hello

2. StringBuffer

Mutable: Thread-safe and synchronized. 

StringBuffer sb = new StringBuffer("Hello");
sb.append(" World");
System.out.println(sb); // Output: Hello World

3. StringBuilder

Mutable: Not thread-safe, but faster than StringBuffer. 

StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");
System.out.println(sb); // Output: Hello World
