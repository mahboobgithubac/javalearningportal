Wrapper Classes in Java

Wrapper classes in Java are used to wrap primitive data types into objects. Java is an object-oriented programming language, and sometimes it requires data to be treated as objects. Wrapper classes provide this capability.

Why Use Wrapper Classes?

  • To use primitive data types in collections (like ArrayList)
  • To use utility methods available in wrapper classes
  • To facilitate autoboxing and unboxing

List of Wrapper Classes

Primitive Type Wrapper Class
byteByte
shortShort
intInteger
longLong
floatFloat
doubleDouble
charCharacter
booleanBoolean

Example 1: Autoboxing and Unboxing

// Autoboxing: converting primitive to object
int a = 10;
Integer obj = a; // Integer.valueOf(a)

// Unboxing: converting object to primitive
int b = obj; // obj.intValue()

System.out.println("Autoboxed: " + obj);
System.out.println("Unboxed: " + b);

Example 2: Using Wrapper with Collections

import java.util.ArrayList;

public class WrapperDemo {
    public static void main(String[] args) {
        ArrayList<Integer> list = new ArrayList<>();
        list.add(100); // Autoboxing
        list.add(200);
        int num = list.get(0); // Unboxing
        System.out.println("First element: " + num);
    }
}

Advantages of Wrapper Classes

  • Support null values
  • Contain utility methods like parseInt, compareTo, etc.
  • Enable use in generic collections

Common Methods

  • Integer.parseInt("123") → returns 123
  • Double.parseDouble("3.14") → returns 3.14
  • Boolean.valueOf("true") → returns true
