Java List and Set Iteration with Tricky Interview Questions

Ways to Iterate List and Set in Java

1. Enhanced For Loop

List<String> list = Arrays.asList("A", "B", "C");
for (String item : list) {
    System.out.println(item);
}

Set<String> set = new HashSet<>(list);
for (String item : set) {
    System.out.println(item);
}

2. Iterator

Iterator<String> listIterator = list.iterator();
while (listIterator.hasNext()) {
    System.out.println(listIterator.next());
}

Iterator<String> setIterator = set.iterator();
while (setIterator.hasNext()) {
    System.out.println(setIterator.next());
}

3. ListIterator (Only for List)

ListIterator<String> listItr = list.listIterator();
while (listItr.hasNext()) {
    System.out.println(listItr.next());
}

üî∏ ListIterator allows bidirectional traversal and modification during iteration.

4. Classic For Loop (Only for List)

for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}

5. Java 8 Stream API

list.stream().forEach(System.out::println);
set.stream().forEach(System.out::println);

6. Java 8 forEach with Lambda

list.forEach(item -> System.out.println(item));
set.forEach(item -> System.out.println(item));

7. Java 8 Parallel Stream

list.parallelStream().forEach(System.out::println);
set.parallelStream().forEach(System.out::println);

8. Enumeration (Legacy)

Vector<String> vector = new Vector<>(list);
Enumeration<String> e = vector.elements();
while (e.hasMoreElements()) {
    System.out.println(e.nextElement());
}

Insert Value at Specific Index in List

List<Integer> numbers = new ArrayList<>(Arrays.asList(10, 20, 30, 35, 40, 50));

// Add 25 at index 3
numbers.add(3, 25);

System.out.println(numbers); // Output: [10, 20, 30, 25, 35, 40, 50]

üî• Tricky Interview Questions on List and Set

1. Can a List/Set contain null?

List<String> list = new ArrayList<>();
list.add(null);
list.add(null); // OK

Set<String> set = new HashSet<>();
set.add(null);
set.add(null); // Still one null

2. What if you modify while iterating?

for (String item : list) {
    list.remove(item); // ‚ùå ConcurrentModificationException
}

‚úÖ Safe way:

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (condition) it.remove();
}

3. Remove duplicates and maintain order

List<Integer> list = Arrays.asList(1, 2, 2, 3, 1);
List<Integer> result = new ArrayList<>(new LinkedHashSet<>(list));
System.out.println(result); // [1, 2, 3]

4. Difference between List.of() vs Arrays.asList()

List<String> list1 = List.of("A", "B");
list1.add("C"); // ‚ùå UnsupportedOperationException

List<String> list2 = Arrays.asList("A", "B");
list2.set(0, "Z"); // ‚úÖ Works
list2.add("C"); // ‚ùå UnsupportedOperationException

5. Find Duplicates in List

Set<String> seen = new HashSet<>();
Set<String> dupes = names.stream()
    .filter(name -> !seen.add(name))
    .collect(Collectors.toSet());

6. Set Order Behavior

- HashSet: No order
- LinkedHashSet: Insertion order
- TreeSet: Sorted

7. Sort Set in Reverse

Set<Integer> set = new HashSet<>(Arrays.asList(5, 2, 9, 1));
List<Integer> sorted = new ArrayList<>(set);
sorted.sort(Comparator.reverseOrder());

8. equals() vs contains()

contains() uses equals() internally. Override equals and hashCode for custom objects.

9. Custom Objects in HashSet

class Book {
    String name;
    Book(String name) { this.name = name; }

    public boolean equals(Object o) {
        return (o instanceof Book) && ((Book) o).name.equals(this.name);
    }

    public int hashCode() {
        return name.hashCode();
    }
}

10. Convert Set to List and Sort

Set<String> set = new HashSet<>(Set.of("B", "A", "C"));
List<String> sortedList = new ArrayList<>(set);
Collections.sort(sortedList);
