Angular Interview Questions and Answers

1. What is Angular?

Angular is a TypeScript-based open-source front-end web application framework developed by Google for building single-page applications (SPA).

2. Difference between AngularJS and Angular (2+)?
  • AngularJS: JavaScript-based, uses controllers, developed before Angular 2.
  • Angular: TypeScript-based, uses components, introduced with Angular 2.
// AngularJS
$scope.message = "Hello";

// Angular (2+)
export class AppComponent {
  message = "Hello";
}
3. What are components in Angular?

Components are the building blocks of Angular applications. Each component controls a part of the screen called a view.

@Component({
  selector: 'app-root',
  template: '<h1>{{title}}</h1>'
})
export class AppComponent {
  title = 'My App';
}
4. What is a module in Angular?

A module is a container for components, directives, pipes, and services.

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  bootstrap: [AppComponent]
})
export class AppModule { }
5. What is data binding in Angular?

Data binding is the mechanism to synchronize data between the model (component) and the view (template).

6. What is interpolation?

Interpolation binds data from the component to the view using double curly braces {{ }}.

<h1>{{title}}</h1>
7. Explain property binding vs event binding.

Property Binding: Binds data from component to DOM property.

<img [src]="imageUrl">

Event Binding: Binds DOM events to component methods.

<button (click)="onClick()">Click Me</button>
8. What is two-way data binding in Angular?

Two-way binding combines property and event binding using [(ngModel)].

<input [(ngModel)]="username">
<p>Welcome {{username}}</p>
9. What are directives? Types of directives?

Directives are classes that add behavior to elements.

  • Component – Directive with a template.
  • Structural – Change DOM structure: *ngIf, *ngFor.
  • Attribute – Change appearance/behavior: ngClass, ngStyle.
10. What is a service in Angular?

A service is a class that holds business logic and reusable data. It can be injected into components.

@Injectable()
export class DataService {
  getData() {
    return ['One', 'Two'];
  }
}
11. What is dependency injection?

Dependency Injection (DI) is a design pattern where components get dependencies from external sources rather than creating them.

constructor(private dataService: DataService) { }
12. What is the purpose of NgModule?

NgModule defines an Angular module. It helps group related components, pipes, directives, and services.

13. What is the Angular CLI? How do you create a project?

Angular CLI is a command-line interface tool to create, manage, and build Angular apps.

npm install -g @angular/cli
ng new my-app
cd my-app
ng serve
14. What is the lifecycle of a component in Angular?

Angular provides lifecycle hooks that give visibility into key events:

  • ngOnInit – after component initialized
  • ngOnChanges – on input changes
  • ngOnDestroy – before component is destroyed
  • Others: ngDoCheck, ngAfterViewInit, etc.
15. What is ngOnInit()? When is it called?

ngOnInit() is a lifecycle hook that is called once after the component is initialized.

export class MyComponent implements OnInit {
  ngOnInit() {
    console.log('Component initialized');
  }
}
