Angular Counter App with Unit Tests

1. Create Angular App

Generate a new Angular project using the CLI:

ng new counter-app
cd counter-app
ng serve

2. Generate Counter Component

Create a new component for the counter functionality:

ng generate component counter

3. Counter Logic (TypeScript)

Implement increment, decrement, and reset logic in counter.component.ts:

import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  templateUrl: './counter.component.html',
  styleUrls: ['./counter.component.css']
})
export class CounterComponent {
  count: number = 0;

  increment(): void {
    this.count++;
  }

  decrement(): void {
    if (this.count > 0) {
      this.count--;
    }
  }

  reset(): void {
    this.count = 0;
  }
}

4. Counter Template (HTML)

Use buttons to control the counter in counter.component.html:

<div style="text-align: center;">
  <h2>Angular Counter App</h2>
  <h3>{{ count }}</h3>
  <button (click)="increment()">Increment</button>
  <button (click)="decrement()">Decrement</button>
  <button (click)="reset()">Reset</button>
</div>

5. Styling (CSS)

Apply simple styling in counter.component.css:

button {
  margin: 0 5px;
  padding: 10px 20px;
  font-size: 16px;
}

6. Use in App Component

Embed the counter component in the main app.component.html:

<app-counter></app-counter>

7. Run the App

Start the development server:

ng serve

Visit http://localhost:4200 to see the app in action.

8. Unit Tests with Jasmine

Write unit tests in counter.component.spec.ts to verify logic:

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { CounterComponent } from './counter.component';

describe('CounterComponent', () => {
  let component: CounterComponent;
  let fixture: ComponentFixture<CounterComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ CounterComponent ]
    }).compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(CounterComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create the component', () => {
    expect(component).toBeTruthy();
  });

  it('should initialize count to 0', () => {
    expect(component.count).toBe(0);
  });

  it('should increment the count', () => {
    component.increment();
    expect(component.count).toBe(1);
  });

  it('should decrement the count', () => {
    component.count = 2;
    component.decrement();
    expect(component.count).toBe(1);
  });

  it('should not decrement below 0', () => {
    component.count = 0;
    component.decrement();
    expect(component.count).toBe(0);
  });

  it('should reset the count to 0', () => {
    component.count = 5;
    component.reset();
    expect(component.count).toBe(0);
  });
});

9. Jasmine vs Karma

Jasmine is the testing framework, and Karma is the test runner used by Angular CLI:

  • Jasmine provides functions like describe, it, and expect.
  • Karma runs the tests in a real browser and shows results.

ng test
