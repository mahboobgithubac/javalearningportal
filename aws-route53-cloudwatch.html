🌐 AWS Route 53 and CloudWatch

1️⃣ AWS Route 53

AWS Route 53 is a highly available and scalable Domain Name System (DNS) web service. It is used to route end users to applications by translating domain names into IP addresses.

🔹 Key Features

  • Domain Registration – Buy and manage domains directly in AWS.
  • DNS Routing – Convert domain names (e.g., example.com) to IP addresses.
  • Health Checks – Monitor application endpoints and reroute traffic if unhealthy.
  • Traffic Policies – Use latency-based, geolocation, or failover routing.
  • Integration with S3 & CloudFront – Point your domain to static sites or CDNs.

🔹 Example Use Case

You have a static website hosted in S3 at mybucket.s3.amazonaws.com. Using Route 53, you can map www.mywebsite.com to this S3 bucket.

2️⃣ AWS CloudWatch

AWS CloudWatch is a monitoring and observability service that collects metrics, logs, and events from AWS services and your applications.

🔹 Key Features

  • Metrics Monitoring – Track CPU, memory, disk, and network usage.
  • Log Management – Store, search, and analyze application logs.
  • Alarms – Trigger notifications or actions when thresholds are breached.
  • Dashboards – Create real-time visualizations of metrics.
  • Integration with AWS Lambda – Automate actions based on events.

🔹 Example Use Case

Monitor an EC2 instance and send an SNS email alert when CPU utilization exceeds 80%.

📌 Summary

  • Route 53 → Manages domains and routes traffic.
  • CloudWatch → Monitors performance, logs, and events.

Together, they ensure your application is reachable and performing optimally.
