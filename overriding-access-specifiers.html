✅ Method Overriding with Different Access Specifiers in Java

When a subclass overrides a method from the superclass, the access specifier of the overriding method cannot be more restrictive than the method in the superclass.

🔒 Rules Summary

Superclass Access Subclass Allowed Valid? Reason
public public ✅ Same access
public protected / default / private ❌ More restrictive
protected protected / public ✅ Same or broader access
protected default / private ❌ More restrictive
default default / protected / public ✅ Same or broader access
private — ✅ Not inherited, so cannot be overridden

🟢 Valid Overriding Example

class Parent {
    protected void show() {
        System.out.println("Parent show()");
    }
}

class Child extends Parent {
    public void show() {
        System.out.println("Child show()");
    }
}

public class Main {
    public static void main(String[] args) {
        Parent obj = new Child();
        obj.show();  // Output: Child show()
    }
}

🔴 Invalid Overriding Example

class Parent {
    public void display() {
        System.out.println("Parent display()");
    }
}

class Child extends Parent {
    protected void display() {  // ❌ Compilation Error
        System.out.println("Child display()");
    }
}

🔒 Private Method (No Overriding)

class Parent {
    private void secret() {
        System.out.println("Parent secret()");
    }
}

class Child extends Parent {
    private void secret() {
        System.out.println("Child secret()");
    }
}

// These are two separate methods, not overriding.

🔁 Overriding with Default Access

class Parent {
    void message() {
        System.out.println("Parent message()");
    }
}

class Child extends Parent {
    public void message() {
        System.out.println("Child message()");
    }
}

This is valid because public is broader than default.
