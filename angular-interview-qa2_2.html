Angular Interview Questions & Answers

1. What is Angular?

Angular is a front-end, open-source web application framework developed and maintained by Google. It's written in TypeScript and allows developers to build dynamic, modern, single-page web applications (SPAs) efficiently.

Angular offers built-in tools like routing, forms handling, HTTP communication, and powerful components and services architecture.

2. Difference between AngularJS and Angular

  • AngularJS (1.x) is based on JavaScript and uses the MVC pattern with scopes and controllers.
  • Angular (2+) is a complete rewrite using TypeScript and introduces modules, components, and services.
// AngularJS
$scope.message = "Hello";

// Angular (2+)
export class AppComponent {
  message = "Hello";
}

3. What are Components in Angular?

Components are the basic building blocks of Angular applications. Each component consists of:

  • A class that holds data and logic
  • An HTML template for the UI
  • Optional CSS for styling
@Component({
  selector: 'app-hello',
  template: '<h2>{{ greeting }}</h2>'
})
export class HelloComponent {
  greeting = 'Hello Angular!';
}

4. What is a Module in Angular?

A module is a container that groups together components, pipes, directives, and services. Every Angular app has at least one root module (AppModule).

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  bootstrap: [AppComponent]
})
export class AppModule { }

5. What is Data Binding in Angular?

Data binding connects the component's logic with the view (template). It keeps the component class and the HTML template in sync.

Angular supports:

  • Interpolation
  • Property binding
  • Event binding
  • Two-way binding

6. What is Interpolation?

Interpolation uses double curly braces {{ }} to display data from the component class into the template.

<p>Welcome, {{ username }}</p>

7. Property Binding vs Event Binding

Property Binding: Used to bind component data to an element's property using [property].

<img [src]="imageUrl">

Event Binding: Used to respond to user actions using (event).

<button (click)="handleClick()">Click Me</button>

8. What is Two-Way Data Binding?

Two-way data binding allows data to flow in both directions — from component to template and vice versa. It uses [(ngModel)].

<input [(ngModel)]="name">
<p>Hello, {{ name }}</p>

9. What are Directives? Types?

Directives are special markers on elements that tell Angular to do something.

  • Structural: Change DOM layout (e.g. *ngIf, *ngFor)
  • Attribute: Change appearance/behavior (e.g. ngClass)
  • Component: A directive with a template
<div *ngIf="isVisible">Visible!</div>

10. What is a Service in Angular?

A service is a reusable class that provides logic, data access, or functions to components. Services promote separation of concerns.

@Injectable({ providedIn: 'root' })
export class UserService {
  getUsers() {
    return ['Alice', 'Bob'];
  }
}

11. What is Dependency Injection?

Dependency Injection (DI) is a design pattern where services are injected into components instead of creating them manually.

constructor(private userService: UserService) { }

12. What is the Purpose of NgModule?

@NgModule defines a compilation context. It organizes components, directives, pipes, and services into cohesive blocks.

13. What is the Angular CLI? How to Create a Project?

Angular CLI is a command-line interface tool for initializing, developing, scaffolding, and maintaining Angular apps.

npm install -g @angular/cli
ng new my-angular-app
cd my-angular-app
ng serve

14. What is the Lifecycle of a Component?

Angular provides lifecycle hooks for component events:

  • ngOnInit – called after constructor and input values are set
  • ngOnChanges – on input property changes
  • ngOnDestroy – just before the component is destroyed

15. What is ngOnInit()? When is it Called?

ngOnInit() is a lifecycle method from the OnInit interface, invoked once after component's first display. It's ideal for initialization logic like data loading.

export class UserComponent implements OnInit {
  ngOnInit() {
    console.log('Component loaded');
  }
}

16. What is Angular Router?

The Angular Router enables navigation between views in a single-page application (SPA). It allows mapping of URL paths to components.

// app-routing.module.ts
const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

17. How do you set up routing in Angular?

Steps to configure routing:

  1. Create app-routing.module.ts
  2. Define Routes
  3. Import RouterModule.forRoot(routes)
  4. Include <router-outlet> in template
@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})

18. What are route guards (CanActivate, CanDeactivate)?

Route guards control access to routes:

  • CanActivate: Checks before route is activated
  • CanDeactivate: Checks before route is left
@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private authService: AuthService) {}

  canActivate(): boolean {
    return this.authService.isLoggedIn();
  }
}

19. How to pass data in routes (params and query params)?

Route Parameters:

{ path: 'user/:id', component: UserComponent }
// Access in component
this.route.params.subscribe(params => {
  console.log(params['id']);
});

Query Parameters:

// Navigate
this.router.navigate(['/search'], { queryParams: { q: 'angular' } });

// Read
this.route.queryParams.subscribe(params => console.log(params['q']));

20. What is Lazy Loading in Angular Routing?

Lazy loading loads modules only when needed using loadChildren.

{ path: 'admin', loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule) }

21. What is the difference between a service and a factory?

Service: Class-based, instantiated by Angular DI.

Factory: Function-based, returns an object (used in AngularJS).

@Injectable()
export class MyService {
  getValue() { return 'Angular'; }
}

22. What is providedIn: 'root' in Angular Services?

This tells Angular to provide the service in the root injector. It makes the service a singleton and avoids manual registration in providers.

@Injectable({ providedIn: 'root' })
export class ApiService { }

23. How does Angular's hierarchical injector work?

Angular's injectors form a hierarchy. Services provided at the component level override the ones provided at the root/module level.

@Component({
  providers: [LocalService]
})

Child components inherit injectors unless overridden.

24. Difference between Template-driven and Reactive forms?

Template-driven Forms: Simpler, defined in the HTML using directives like ngModel. Ideal for simple forms.

Reactive Forms: Model-driven, defined in the component using FormGroup/FormControl. Better for complex forms and validations.

// Template-driven
<input [(ngModel)]=\"name\" name=\"name\" required />

// Reactive
this.form = new FormGroup({
  name: new FormControl('', Validators.required)
});

25. How do you create a reactive form in Angular?

Use FormGroup and FormControl in the component and bind using formGroup and formControlName.

// Component
this.loginForm = new FormGroup({
  email: new FormControl(''),
  password: new FormControl('')
});

// Template
<form [formGroup]=\"loginForm\">
  <input formControlName=\"email\" />
  <input formControlName=\"password\" type=\"password\" />
</form>

26. What is FormBuilder in Angular?

FormBuilder is a service that simplifies the syntax to create forms.

constructor(private fb: FormBuilder) {}

this.loginForm = this.fb.group({
  email: [''],
  password: ['']
});

27. How do you validate a form?

Use built-in validators like Validators.required, Validators.email etc., while creating form controls.

this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', Validators.required]
});

Then check in template:

<div *ngIf=\"form.controls.email.errors?.required\">
  Email is required.
</div>

28. What is Change Detection in Angular?

Change Detection is the mechanism Angular uses to track changes in component state and update the DOM accordingly. Angular checks bindings and re-renders the view when data changes.

29. Difference between Default and OnPush change detection strategy?

  • Default: Detects changes on every event cycle (may affect performance).
  • OnPush: Only detects when input properties change or a manual trigger occurs.
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush
})

30. How do you optimize performance in large Angular applications?

  • Use ChangeDetectionStrategy.OnPush
  • Lazy load modules
  • TrackBy in *ngFor to avoid re-rendering
  • Unsubscribe from observables
  • Use Pure Pipes and async pipe

31. What are Observables?

Observables are a core concept of RxJS used for handling async data streams (e.g., HTTP requests, events).

const obs$ = new Observable(observer => {
  observer.next('Hello');
  observer.complete();
});
obs$.subscribe(data => console.log(data));

32. Difference between Observable and Promise?

  • Observable: Multiple values over time, cancellable, lazy.
  • Promise: One-time event, not cancellable.

33. What are common RxJS operators?

  • map – Transform values
  • switchMap – Cancel previous observable
  • mergeMap – Merge multiple streams
  • catchError – Error handling
this.user$
  .pipe(
    switchMap(id => this.api.getUser(id)),
    catchError(err => of(null))
  );

34. What is a Subject and BehaviorSubject?

  • Subject: Multicast observable with manual emissions
  • BehaviorSubject: Holds last emitted value
const subject = new BehaviorSubject(0);
subject.subscribe(console.log); // 0
subject.next(1);

35. How do you handle async operations using RxJS?

Use HTTPClient + RxJS operators like switchMap, catchError, finalize.

this.http.get('/api/users')
  .pipe(
    map(users => users.filter(u => u.active)),
    catchError(err => of([]))
  );

36. What is a dynamic component in Angular?

It is a component created at runtime using ViewContainerRef.createComponent().

@ViewChild('container', { read: ViewContainerRef }) container: ViewContainerRef;

ngAfterViewInit() {
  this.container.createComponent(MyDynamicComponent);
}

37. How do you use ng-content and content projection?

Used to project content into a component's template.

// Parent usage
<app-card><p>Projected!</p></app-card>

// app-card.component.html
<ng-content></ng-content>

38. What is a custom pipe and how to create one?

Custom pipes transform values in templates.

@Pipe({ name: 'capitalize' })
export class CapitalizePipe implements PipeTransform {
  transform(value: string) {
    return value.charAt(0).toUpperCase() + value.slice(1);
  }
}

Use: {{ 'hello' | capitalize }}

39. How do you handle error handling globally in Angular?

Use ErrorHandler class globally:

@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  handleError(error: any) {
    console.error('Global Error:', error);
  }
}
providers: [{ provide: ErrorHandler, useClass: GlobalErrorHandler }]

40. What is Angular Universal?

Angular Universal enables server-side rendering (SSR) of Angular apps to improve SEO and performance.

41. What are interceptors? How do you use them?

Interceptors modify HTTP requests/responses globally using HttpInterceptor.

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler) {
    const authReq = req.clone({ setHeaders: { Authorization: 'Bearer TOKEN' }});
    return next.handle(authReq);
  }
}

42. Explain Angular compiler types: JIT vs AOT

  • JIT (Just-in-Time): Compiles in browser at runtime (dev mode)
  • AOT (Ahead-of-Time): Compiles at build time (prod)

43. What is Ahead-of-Time (AOT) Compilation?

AOT pre-compiles HTML and TypeScript to JS before running in the browser, improving performance and catching template errors earlier.

44. What are pure and impure pipes?

Pure Pipe: Executes only on changed input.

Impure Pipe: Executes on every change detection cycle.

@Pipe({ name: 'myPipe', pure: false })

45. How do you test Angular components and services?

Use Jasmine + Karma. For components, use TestBed and simulate interactions with fixture.

46. What is TestBed in Angular testing?

TestBed configures the testing module, allowing testing of Angular components in isolation.

beforeEach(() => {
  TestBed.configureTestingModule({
    declarations: [MyComponent],
    providers: [MyService]
  }).compileComponents();
});

47. How do you write unit tests for Angular services using Jasmine/Karma?

describe('MyService', () => {
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({ providers: [MyService] });
    service = TestBed.inject(MyService);
  });

  it('should return expected value', () => {
    expect(service.getValue()).toBe('Angular');
  });
});

48. How do you share data between components?

You can share data between Angular components in several ways:

  • Parent to Child: Use @Input()
  • Child to Parent: Use @Output() with EventEmitter
  • Unrelated Components: Use a shared service with RxJS BehaviorSubject
@Injectable({ providedIn: 'root' })
export class SharedService {
  private messageSource = new BehaviorSubject<string>('default message');
  currentMessage = this.messageSource.asObservable();

  changeMessage(message: string) {
    this.messageSource.next(message);
  }
}

49. What is the purpose of ngZone?

NgZone is used to run code outside Angular’s change detection to improve performance.

constructor(private ngZone: NgZone) {}

runOutsideAngularZone() {
  this.ngZone.runOutsideAngular(() => {
    setTimeout(() => {
      this.ngZone.run(() => {
        console.log('Back in Angular zone');
      });
    }, 1000);
  });
}

50. How to implement authentication and route protection in Angular?

Use an AuthService to manage login and AuthGuard with CanActivate to protect routes.

@Injectable({ providedIn: 'root' })
export class AuthService {
  isAuthenticated(): boolean {
    return !!localStorage.getItem('token');
  }
}

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private auth: AuthService, private router: Router) {}

  canActivate(): boolean {
    if (this.auth.isAuthenticated()) return true;
    this.router.navigate(['/login']);
    return false;
  }
}

// app-routing.module.ts
const routes: Routes = [
  { path: 'dashboard', component: DashboardComponent, canActivate: [AuthGuard] },
  { path: 'login', component: LoginComponent }
];

51. How do you use environment variables in Angular?

Angular uses environment.ts and environment.prod.ts to manage configuration.

// environment.ts
export const environment = {
  production: false,
  apiUrl: 'http://localhost:3000/api'
};

// usage
this.http.get(environment.apiUrl + '/users');

52. How do you handle HTTP calls and APIs?

Use HttpClient and handle responses with RxJS operators like map and catchError.

@Injectable({ providedIn: 'root' })
export class ApiService {
  constructor(private http: HttpClient) {}

  getUsers() {
    return this.http.get('/api/users').pipe(
      map(data => data),
      catchError(error => {
        console.error('API error:', error);
        return throwError(() => error);
      })
    );
  }
}

53. How would you organize a large Angular project?

  • CoreModule: singleton services, guards, interceptors
  • SharedModule: reusable components and pipes
  • Feature modules: domain-specific organization
  • Use a folder-by-feature structure
src/
├── app/
│   ├── core/              # singleton services
│   ├── shared/            # reusable modules
│   ├── features/          # each feature
│   │   ├── auth/
│   │   ├── dashboard/
│   │   └── products/
│   └── app.module.ts
