Performance Tuning in Hibernate / JPA

1. Fetch Strategy
TypeUse CaseExample
LAZY Best for performance, loads on access @OneToMany(fetch = FetchType.LAZY)
EAGER Use only if you always need the data @ManyToOne(fetch = FetchType.EAGER)
2. Avoid N+1 Problem

Problem:

List<Department> departments = repo.findAll();
for (Department dept : departments) {
    System.out.println(dept.getEmployees().size());
}

Solution:

  • Using JOIN FETCH:
    • @Query("SELECT d FROM Department d JOIN FETCH d.employees")
List<Department> findAllWithEmployees();
  • Using @EntityGraph:
    • @EntityGraph(attributePaths = "employees")
List<Department> findAll();
3. Batching for Inserts and Updates 
# application.properties
spring.jpa.properties.hibernate.jdbc.batch_size=30
spring.jpa.properties.hibernate.order_inserts=true
spring.jpa.properties.hibernate.order_updates=true

⚠️ Works only in same session/transaction.

4. Enable Second-Level Cache (L2)
  1. Add EhCache or Caffeine to pom.xml:
    2. <dependency>
  <groupId>org.ehcache</groupId>
  <artifactId>ehcache</artifactId>
</dependency>
  2. Enable in entity:
    3. @Cacheable
@org.hibernate.annotations.Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
  3. Enable config:
    4. spring.jpa.properties.hibernate.cache.use_second_level_cache=true
spring.jpa.properties.hibernate.cache.region.factory_class=org.hibernate.cache.jcache.JCacheRegionFactory
5. Pagination Instead of Fetching All 
Page<Product> products = productRepo.findAll(PageRequest.of(0, 10));
6. Use Indexing on Columns

Index columns used in WHERE, JOIN, ORDER BY:

CREATE INDEX idx_product_category ON product(category_id);
7. DTO Projections 
public interface ProductDTO {
  String getTitle();
  Double getPrice();
}
List<ProductDTO> findByCategoryId(Long categoryId);
8. Use @Modifying for Bulk Updates 
@Modifying
@Query("UPDATE Product p SET p.price = :price WHERE p.id = :id")
void updatePriceById(@Param("price") double price, @Param("id") Long id);
9. Avoid CascadeType.ALL on Large Collections

Prefer specific cascades like CascadeType.PERSIST or MERGE.

10. Use Read-Only Queries 
@QueryHints(@QueryHint(name = "org.hibernate.readOnly", value = "true"))
@Query("SELECT p FROM Product p WHERE p.id = :id")
Product findReadOnly(@Param("id") Long id);
11. Use StatelessSession for Bulk Processing 
SessionFactory sf = entityManagerFactory.unwrap(SessionFactory.class);
StatelessSession session = sf.openStatelessSession();
12. Tune Connection Pool (HikariCP) 
spring.datasource.hikari.maximum-pool-size=20
spring.datasource.hikari.minimum-idle=5
spring.datasource.hikari.idle-timeout=30000
spring.datasource.hikari.connection-timeout=20000
13. Enable Hibernate Statistics (for Debugging) 
spring.jpa.properties.hibernate.generate_statistics=true
