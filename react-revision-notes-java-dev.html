React Quick Revision Notes

For Java Developers — Light Mode

1. React Basics

  • What is React? A JS library for building UIs using components and declarative syntax (JSX).
  • Key ideas: Component-based, Virtual DOM, unidirectional data flow.

2. Core Concepts

  • Components: Functional (hooks) vs Class (legacy).
  • Props vs State: Props = read-only inputs; State = component-local data.
  • Virtual DOM: Efficient diffing & minimal real DOM updates.

3. Hooks (must-know)

  • useState — local state
  • useEffect — side-effects & lifecycle
  • useContext — access shared data
  • useRef — mutable refs without re-render
  • useReducer — complex state management

Be ready to explain when to choose useReducer over useState.

4. Lifecycle Mapping

  • Class: componentDidMount, componentDidUpdate, componentWillUnmount
  • Function: useEffect(() => { ... }, [deps]) and cleanup via return function.

5. Communication

  • Parent → Child: props
  • Child → Parent: callbacks passed via props
  • Global state: Context API, Redux

6. Lists & Conditional Rendering

  • Use array.map() to render lists and provide a stable key.
  • Conditionals: && and ternary cond ? A : B.

7. Forms

  • Controlled components: input value bound to state.
  • Uncontrolled components: use ref to read DOM value.

8. React Router

  • Use BrowserRouter, Routes, Route.
  • Link for navigation; useNavigate() for programmatic redirects.

9. State Management

  • Local: useState, useReducer.
  • Global: Context API, Redux (enterprise apps).

10. Performance

  • React.memo to avoid re-renders.
  • useCallback for stable function references.
  • useMemo to memoize expensive calculations.
  • Code-splitting: React.lazy + Suspense.

11. Backend Integration (Spring Boot)

  • Call APIs via fetch or axios inside useEffect.
  • Handle CORS on the server (global config or controller-level).
  • Auth: prefer HTTP-only cookies for JWT in production; mitigate XSS/CSRF.

12. Common Interview Questions

  1. Functional vs Class components?
  2. Virtual DOM vs Real DOM?
  3. How does useEffect map to lifecycle methods?
  4. State vs Props?
  5. When to use Redux vs Context API?

  7. What is "lifting state up"?
  8. Controlled vs Uncontrolled components?
  9. How to improve performance in React?
  10. How to secure React ↔ Spring Boot integration?

Java Analogies (Quick Memory Hooks)

  • Props ≈ method parameters
  • State ≈ instance fields
  • Context/Redux ≈ application-scoped singleton bean
  • useEffect ≈ @PostConstruct/@PreDestroy lifecycles
  • React Router ≈ Spring MVC routing (URL ↔ component mapping)

Fast Snippets

Basic fetch inside useEffect

useEffect(() => {
  let cancelled = false;
  (async () => {
    try {
      const res = await fetch('/api/products');
      if (!res.ok) throw new Error('Network error');
      const data = await res.json();
      if (!cancelled) setProducts(data);
    } catch (e) { setError(e.message); }
  })();
  return () => { cancelled = true; };
}, []);

Memoization

const expensive = useMemo(() => compute(items), [items]);
const onClick = useCallback(() => doSomething(expensive), [expensive]);
