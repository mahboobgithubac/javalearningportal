Deploy Spring Boot REST API to AWS

1. Prerequisites

  • AWS account
  • AWS CLI installed and configured (aws configure)
  • Spring Boot application ready to deploy
  • Maven or Gradle installed

2. Package the Application

mvn clean package
This generates a JAR file in the target/ folder.

3. Configure Spring Boot for AWS Deployment

Typically, you don’t need to make large changes to your Spring Boot app to run on AWS. However, you might need:

  • Externalized Configuration — use application.properties or environment variables for DB URLs, S3 credentials, etc.
  • Server Port — If deploying to Elastic Beanstalk or ECS, ensure: 
    server.port=${PORT:5000}
  • Logging — Use CloudWatch-compatible logging patterns.
  • Database Connection — If using RDS: 
    spring.datasource.url=jdbc:mysql://${DB_HOST}:${DB_PORT}/${DB_NAME}
spring.datasource.username=${DB_USER}
spring.datasource.password=${DB_PASS}

4. AWS Deployment Options

Option 1: AWS Elastic Beanstalk

  1. Create an Elastic Beanstalk application in the AWS Console.
  2. Upload your .jar file.
  3. AWS will automatically create an EC2 environment to run it.

Option 2: EC2 Instance

  1. Launch an EC2 instance (Amazon Linux / Ubuntu).
  2. Install Java: 
    sudo amazon-linux-extras install java-openjdk17
  3. Copy your JAR file to EC2: 
    scp target/myapp.jar ec2-user@<EC2-IP>:/home/ec2-user/
  4. Run your app: 
    java -jar myapp.jar

Option 3: AWS ECS (Docker)

  1. Create a Dockerfile for your Spring Boot app.
  2. Push the Docker image to Amazon ECR.
  3. Deploy via ECS service and Fargate.

5. Domain & SSL

  • Use Route 53 to configure your domain.
  • Use AWS Certificate Manager (ACM) for free SSL.
  • Attach the certificate to your Load Balancer.

6. Security Best Practices

  • Use IAM Roles instead of hardcoding AWS keys.
  • Enable HTTPS for all endpoints.
  • Enable security groups and restrict inbound traffic.
