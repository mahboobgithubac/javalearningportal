React Interview Questions and Answers

1. What are the features of React?

  • Component-Based: UI is divided into reusable components.
  • JSX: Syntax extension that allows writing HTML inside JavaScript.
  • Virtual DOM: Efficiently updates the real DOM using a diffing algorithm.
  • Unidirectional Data Flow: Data flows from parent to child, making it predictable.
  • Declarative: UI changes automatically when state or props change.
  • React Native: Allows building mobile apps using React.

2. What is JSX?

JSX (JavaScript XML) is a syntax extension for JavaScript that allows writing HTML-like code inside JavaScript. It makes code easier to write and understand in React. Example: 
const element = <h1>Hello, World!</h1>;

3. Can web browsers read JSX directly?

No, web browsers cannot read JSX directly. JSX must be transpiled into regular JavaScript using tools like Babel before it can run in the browser.

4. What is the Virtual DOM?

The Virtual DOM is a lightweight copy of the real DOM. When the state of a component changes, React creates a new Virtual DOM, compares it with the previous one (diffing), and updates only the changed elements in the real DOM (reconciliation). This makes updates faster and more efficient.

5. Why use React instead of other frameworks, like Angular?

React offers several advantages over other frameworks like Angular:
  • Simplicity: React is easier to learn due to its simpler component model.
  • Flexibility: React focuses on the view layer only, giving developers more flexibility with libraries.
  • Performance: Virtual DOM improves performance by minimizing direct DOM manipulations.
  • Community and Ecosystem: Huge community support, lots of libraries and tools available.
  • Reusable Components: Promotes code reusability and modular architecture.

6. What is the difference between the ES6 and ES5 standards?

ES6 (ECMAScript 2015) introduced many new features compared to ES5, such as:
  • Let & Const: New ways to declare variables with block scope.
  • Arrow Functions: Shorter syntax for functions: (a) => a * 2
  • Classes: Native class syntax in JavaScript.
  • Modules: import and export support.
  • Template Literals: Multi-line and embedded expressions with backticks.
  • Promises: Built-in support for asynchronous operations.

7. How do you create a React app?

You can create a new React app using Create React App (CRA) by running: 
npx create-react-app my-app
This sets up a React project with all necessary build tools and configuration.

8. What is an event in React?

An event in React is an action that a user or browser can trigger, such as clicking, submitting a form, or pressing a key. React handles events using camelCase syntax and JSX.

9. How do you create an event in React?

Events are created by passing a function to an event handler attribute in JSX: 

<button onClick={handleClick}>Click Me</button>

function handleClick() {
  alert('Button clicked!');
}

10. What are synthetic events in React?

Synthetic events are React's cross-browser wrapper around native DOM events. They provide a consistent interface regardless of browser differences and work similarly to native events.

11. Explain how lists work in React.

Lists in React are created using the JavaScript map() function to iterate over an array and return a list of JSX elements: 

const items = ['Apple', 'Banana', 'Mango'];
const listItems = items.map(item => <li>{item}</li>);
Rendered inside JSX as: <ul>{listItems}</ul>

12. Why is there a need for using keys in Lists?

Keys help React identify which items have changed, are added, or removed. They improve performance by enabling efficient re-rendering. Keys should be unique and stable (e.g., IDs).

13. What are forms in React?

Forms in React are used to handle user input. React uses controlled components where form data is handled by React's state: 

const [name, setName] = useState('');

<form onSubmit={handleSubmit}>
  <input value={name} onChange={e => setName(e.target.value)} />
  <button type="submit">Submit</button>
</form>

14. How do you create forms in React?

Forms in React are created using controlled components, where input values are controlled by state: 

import { useState } from 'react';

function MyForm() {
  const [name, setName] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    alert(name);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={name} onChange={(e) => setName(e.target.value)} />
      <button type="submit">Submit</button>
    </form>
  );
}

15. How do you write comments in React?

In JSX, comments are written inside curly braces using the {/* comment */} syntax: 

return (
  <div>
    {/* This is a comment */}
    <p>Hello React</p>
  </div>

16. What is an arrow function and how is it used in React?

An arrow function is a concise way to write functions in JavaScript, introduced in ES6: 
const greet = (name) => 'Hello, ' + name;
In React, arrow functions are commonly used for event handlers and functional components: 

const MyComponent = () => {
  const handleClick = () => alert('Clicked!');
  return <button onClick={handleClick}>Click Me</button>;
};
Arrow functions do not bind their own this, making them useful in class components as well.

17. How is React different from React Native?

  • React: Used for building web applications (runs in the browser).
  • React Native: Used for building native mobile applications for iOS and Android.
  • Rendering: React uses HTML and CSS; React Native uses native components like View, Text, and StyleSheet.
  • Platform: React runs in a browser; React Native compiles to mobile code using native APIs.

18. How is React different from Angular?

  • Type: React is a library; Angular is a full-fledged framework.
  • Language: React uses JavaScript/JSX; Angular uses TypeScript.
  • Data Binding: React uses one-way data binding; Angular supports two-way data binding.
  • DOM: React uses virtual DOM; Angular uses real DOM with change detection.
  • Learning Curve: React is easier and more flexible; Angular has a steeper learning curve due to its complexity.

19. What are the components in React?

Components are the building blocks of a React application. They are reusable, independent pieces of UI. There are two types:
  • Functional Components: JavaScript functions that return JSX.
  • Class Components: ES6 classes that extend React.Component and include a render method.

20. What is the use of render() in React?

The render() method is used in class components to describe what UI should look like. It returns JSX, which React renders into the DOM. 

class MyComponent extends React.Component {
  render() {
    return <h1>Hello, World!</h1>;
  }
}

21. What is a state in React?

State is a built-in object used to store data that changes over time and affects how a component renders. Each component can maintain its own state.

22. How do you implement state in React?

In functional components using the useState hook: 

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
In class components: 

class Counter extends React.Component {
  constructor() {
    super();
    this.state = { count: 0 };
  }

  render() {
    return <button onClick={() => this.setState({ count: this.state.count + 1 })}>
      {this.state.count}
    </button>;
  }
}

23. How do you update the state of a component?

You update state using the setState() method in class components or the setStateFunction from the useState hook in functional components. 

const [name, setName] = useState("John");
setName("Alice"); // updates state
      

this.setState({ name: "Alice" });

24. What are props in React?

Props (short for "properties") are read-only inputs passed from a parent component to a child component. They allow data to flow from parent to child.

25. How do you pass props between components?

Props are passed like HTML attributes: 

function Child(props) {
  return <p>Hello, {props.name}</p>;
}

function Parent() {
  return <Child name="Alice" />;
}

27. What is a higher-order component in React?

A Higher-Order Component (HOC) is a function that takes a component and returns a new component with added functionality. It’s commonly used for code reuse. 

const withLogger = (WrappedComponent) => {
  return function(props) {
    console.log("Props:", props);
    return <WrappedComponent {...props} />;
  };
};

const MyComponentWithLogger = withLogger(MyComponent);

30. Explain the lifecycle methods of components.

Lifecycle methods are only available in class components and define different stages of a component's life:
  • Mounting: When the component is being inserted into the DOM.
    • constructor()
    • static getDerivedStateFromProps()
    • render()
    • componentDidMount()
  • Updating: When the component is being re-rendered due to changes in props or state.
    • static getDerivedStateFromProps()
    • shouldComponentUpdate()
    • render()
    • getSnapshotBeforeUpdate()
    • componentDidUpdate()
  • Unmounting: When the component is being removed from the DOM.
    • componentWillUnmount()
  • Error Handling: If there is an error during rendering.
    • componentDidCatch()
    • static getDerivedStateFromError()

31. What is Redux?

Redux is a predictable state container for JavaScript applications, commonly used with React for managing application state in a centralized way. It allows different components to access and update shared state efficiently using a unidirectional data flow.

32. What are the components of Redux?

The main components of Redux are:
  • Store: Holds the entire state of the application.
  • Actions: Plain JavaScript objects that describe what happened (e.g., `{ type: 'INCREMENT' }`).
  • Reducers: Pure functions that determine how the state changes in response to actions.
  • Dispatch: A method used to send actions to the store.
  • Subscribers: Functions that listen for state changes.

33. What is the Flux?

Flux is an application architecture developed by Facebook for building client-side web applications. It emphasizes a unidirectional data flow, similar to Redux. It consists of:
  • Dispatcher: A central hub that manages all data flow.
  • Stores: Contain application state and logic.
  • Views: React components that listen for store changes.
  • Actions: Payloads of information sent to the dispatcher.

34. How is Redux different from Flux?

Flux Redux
Uses multiple stores Uses a single centralized store
Has a dispatcher for dispatching actions No dispatcher; uses reducers instead
Stores have mutable state State is immutable; returns a new state each time
More boilerplate and less opinionated Simpler and more predictable with strict design

36. Why do we need React Router?

React is a single-page application (SPA) framework. Without routing, the entire page would reload on every URL change. React Router provides dynamic routing so the view changes without reloading the page, enhancing the user experience and performance.

37. How is React routing different from conventional routing?

  • Conventional Routing: A request is sent to the server for every route change, and the server responds with a new HTML page.
  • React Routing: Uses client-side routing. Only the content component changes on route change, without reloading the page.

38. How do you implement React routing?

You can implement React routing using react-router-dom: 
npm install react-router-dom
Example: 

import { BrowserRouter as Router, Routes, Route, Link } from 'react-router-dom';

  • Inline styling: Using the style attribute with a JavaScript object.
  • CSS stylesheets: Importing and using external CSS files.
  • CSS Modules: Scoped CSS files imported as JavaScript objects.
  • Styled-components: Using a library to write actual CSS in JS with component-level scope.

// Inline style example

40. Explain the use of CSS modules in React.

CSS Modules allow writing CSS that is scoped locally to the component, avoiding global conflicts. File names typically end with .module.css.

Steps:
  1. Create a CSS file: styles.module.css
  2. Import it: import styles from './styles.module.css';
  3. Apply styles: <div className={styles.container}></div>

// styles.module.css
.container {
  color: blue;
  font-weight: bold;
}

// Component.js
import styles from './styles.module.css';

function MyComponent() {
  return <div className={styles.container}>Styled with CSS Module</div>;
}

41. What is the use of shouldComponentUpdate()?

shouldComponentUpdate() is a lifecycle method used in class components to determine whether the component should re-render or not. It helps improve performance by preventing unnecessary renders. 

shouldComponentUpdate(nextProps, nextState) {
  return nextProps.value !== this.props.value;
}

42. What is the difference between useState() and useEffect()?

  • useState() is used to create and manage state in a functional component.
  • useEffect() is used to perform side effects (like fetching data, updating DOM, etc.) after render.

43. How does useContext() work?

useContext() is a hook that allows functional components to access the value of a React Context without using a Context.Consumer. 

const ThemeContext = React.createContext('light');
const theme = useContext(ThemeContext);

44. What is useRef() and how is it used?

useRef() returns a mutable object that persists across renders. It's commonly used to access DOM elements or store mutable values that don't trigger re-render. 

const inputRef = useRef(null);

useEffect(() => {
  inputRef.current.focus();
}, []);

return <input ref={inputRef} />;

45. What is useMemo() and why is it useful?

useMemo() memoizes a computed value so it is recalculated only when its dependencies change. It helps optimize performance. 

const expensiveValue = useMemo(() => {
  return computeExpensiveValue(a, b);
}, [a, b]);

46. What is useCallback() and when should you use it?

useCallback() memoizes a callback function so it does not get re-created on every render unless its dependencies change. It's useful when passing callbacks to child components to avoid unnecessary re-renders. 

const handleClick = useCallback(() => {
  console.log('Clicked!');
}, []);

47. What is the difference between Context API and Redux?

  • Context API is a built-in React feature used for simple state sharing across components without prop drilling.
  • Redux is an external library that provides a more powerful state management solution with middleware, dev tools, and strict patterns.
  • Use Context API for light, small apps and Redux for complex state needs and scalable architecture.

48. What is a selector in Redux?

A selector is a function that extracts or derives specific parts of the state from the Redux store. 

const selectUser = (state) => state.user;
const userName = useSelector(selectUser).name;
It helps decouple components from the shape of the Redux store and improves reusability.

49. Explain the flow of data in Redux (actions, reducers, store).

  1. Action: A plain object describing what happened.
  2. Reducer: A function that receives the current state and the action, and returns a new state.
  3. Store: Holds the application state and allows access via getState() and updates via dispatch(action).

// Action
const increment = () => ({ type: 'INCREMENT' });

// Reducer
const counter = (state = 0, action) => {
  switch(action.type) {
    case 'INCREMENT': return state + 1;
    default: return state;
  }
};

50. How can you optimize performance in a React app?

  • Use React.memo() for pure functional components.
  • Use useMemo() and useCallback() to memoize expensive functions and handlers.
  • Split code with React.lazy and Suspense.
  • Avoid unnecessary state or props updates.
  • Use keys properly in lists.
  • Minimize re-renders using shouldComponentUpdate or React.PureComponent.

51. What is React.memo()?

React.memo() is a higher-order component that prevents re-rendering if the props have not changed. It works similar to PureComponent but for functional components. 

const MyComponent = React.memo(function MyComponent(props) {
  // renders only when props change
});

52. How does key prop help in performance?

key helps React identify which items have changed, are added, or are removed. It allows efficient re-rendering by minimizing DOM updates. 

{items.map(item => <li key={item.id}>{item.name}</li>)}

53. What is lazy loading in React?

Lazy loading is a technique where components are loaded only when needed, improving performance and initial load time. 

const LazyComponent = React.lazy(() => import('./LazyComponent'));

<Suspense fallback={"Loading..."}>
  <LazyComponent />
</Suspense>

54. How do you prevent unnecessary re-renders?

  • Use React.memo to memoize components.
  • Use useCallback to memoize functions.
  • Use useMemo for expensive computations.
  • Split large components into smaller ones.
  • Minimize the use of anonymous functions in render.
  • Avoid changing props or state unnecessarily.

84. What is reconciliation in React?
Reconciliation is the process React uses to update the DOM when a component's state or props change. It compares the new virtual DOM with the previous one and efficiently updates only the changed parts using a diffing algorithm.
85. What is the difference between React 17 and 18?
React 17: Focused on gradual upgrades and better event delegation.
React 18: Introduced concurrent rendering, automatic batching, and features like startTransition, Suspense for data fetching, and useDeferredValue.
86. Explain concurrent rendering in React 18.
Concurrent rendering allows React to interrupt rendering tasks to handle higher priority updates. It improves user experience by making the UI more responsive during large or complex updates.

Example using startTransition:

  setState(value); // non-urgent update
});

88. What is hydration in React?
