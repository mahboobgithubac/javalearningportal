üîç What Are Private Interface Methods?

In Java 9 and later, interfaces can include private methods that are used to share code between default or static methods inside the interface.

‚úÖ Why Were They Introduced?

Before Java 9, shared logic between default/static methods had to be duplicated. Java 9 allows private methods inside interfaces to avoid redundancy and improve code reusability.

üìå Types of Private Interface Methods

Modifier Usage Accessible From
private Reusable helper for default methods Only inside interface
private static Reusable helper for static methods Only inside interface

‚úÖ Example: Using Private Interface Methods

public interface MyInterface {

    default void displayA() {
        log("A");
    }

    default void displayB() {
        log("B");
    }

    private void log(String msg) {
        System.out.println("Logging: " + msg);
    }

    static void staticDisplay() {
        staticLog("Static method called");
    }

    private static void staticLog(String msg) {
        System.out.println("Static Log: " + msg);
    }
}
public class MyClass implements MyInterface {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj.displayA(); // Logging: A
        obj.displayB(); // Logging: B
        MyInterface.staticDisplay(); // Static Log: Static method called
    }
}

üß† Key Points

  • Private methods are only for internal use in the interface.
  • They help eliminate code duplication among default or static methods.
  • Can't be abstract or accessed by implementing classes.
