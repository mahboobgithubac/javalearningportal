Servlet Interview Questions and Answers

Essential Servlet questions for Java backend developers

1. What is a Servlet?

Answer: A Servlet is a Java class used to handle requests and responses in a web application. It runs on a servlet container like Apache Tomcat and follows the `javax.servlet` API to handle HTTP requests (GET, POST, etc.).

2. Lifecycle methods of a Servlet?

Answer: A servlet has the following lifecycle methods:

  • init() - Called once when the servlet is initialized.
  • service() - Called for every request.
  • destroy() - Called before the servlet is unloaded.

3. Difference between `doGet()` and `doPost()`?

Answer:

  • doGet: Used to request data; data is sent in the URL (visible).
  • doPost: Used to send data; data is sent in the request body (hidden).

4. What is the role of the web.xml file?

Answer: `web.xml` is the deployment descriptor used to configure servlets, servlet-mappings, welcome files, error pages, filters, and more. Itâ€™s located in the WEB-INF directory of a web application.

5. What are ServletConfig and ServletContext?

ServletConfig: Object used to pass initialization parameters to a single servlet.

ServletContext: Object used to share data across the entire application (all servlets).

6. How to handle session management in Servlets?

Answer: Sessions in servlets can be managed using:

  • Cookies
  • URL Rewriting
  • Hidden Form Fields
  • HttpSession API (most common)

7. How do you redirect a request in Servlet?

Answer: Using response.sendRedirect("newPage.jsp"). This tells the client browser to make a new request to the provided URL.

8. How do you forward a request to another resource?

Answer: Use RequestDispatcher:

RequestDispatcher rd = request.getRequestDispatcher("another.jsp");
rd.forward(request, response);

9. Can a servlet have multiple mappings?

Answer: Yes. A single servlet class can be mapped to multiple URL patterns in `web.xml` or using `@WebServlet(urlPatterns = {"/a", "/b"})` annotation.

10. How to upload a file using Servlet?

Answer: You can use Apache Commons FileUpload or Servlet 3.0's built-in multipart support:

@MultipartConfig
public class UploadServlet extends HttpServlet {
  protected void doPost(HttpServletRequest request, HttpServletResponse response) {
    Part filePart = request.getPart("file");
    String fileName = filePart.getSubmittedFileName();
    filePart.write("uploads/" + fileName);
  }
}
