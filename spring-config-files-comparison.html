âœ… If Both Files Exist: Which One Wins?

Spring Boot loads both application.yml and application.properties, but if the same key is defined in both files, application.properties takes precedence.

Example:

application.yml:

server:
  port: 8080

application.properties:

server.port=9090
âœ… The application will run on port 9090 (from application.properties).

âœ… Best Practice

  • Avoid defining the same keys in both files.
  • Stick to one format consistently throughout your project.

ðŸ”§ How Spring Boot Loads Configurations

Spring Boot uses a defined order when loading config files (via ConfigFileApplicationListener):

  1. application.properties (highest file priority)
  2. application.yml
  3. Profile-specific files like application-dev.properties
  4. External sources (e.g., command-line args, env vars)
