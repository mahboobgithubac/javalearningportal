Marker Interface in Java

A Marker Interface in Java is an interface that contains no methods or fields. It is used to convey metadata or "mark" a class to signal some special behavior to the JVM or frameworks.

Common Marker Interfaces in Java

  • java.io.Serializable
  • java.lang.Cloneable
  • java.util.RandomAccess
  • javax.ejb.Remote

Why Use Marker Interface?

  • To provide run-time information to the JVM.
  • To indicate permission for certain operations.
  • Used by frameworks or libraries to provide conditional behavior.

Example: Marker Interface

interface MyMarker { 
    // no methods 
}

class MyClass implements MyMarker {
    public void show() {
        System.out.println("Marked class logic.");
    }
}

public class MarkerDemo {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        if (obj instanceof MyMarker) {
            obj.show(); // Only runs if object is "marked"
        }
    }
}

Alternative to Marker Interface

Java annotations (e.g., @Override, @Deprecated) are now commonly used as a more powerful alternative to marker interfaces.
