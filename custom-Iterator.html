Custom Iterator in Java – Quick Guide & Example

Java • Collections • Iterator

What is an Iterator?

An Iterator<T> lets you traverse elements of a collection one by one. You can create a custom iterator when you need custom traversal logic (e.g., reverse order, filtering, skipping).

Required Methods

  • boolean hasNext() – Are there more elements?
  • T next() – Return the next element.
  • Optional: void remove() – Usually unsupported.

Typical Steps

  1. Implement Iterator<T>.
  2. Maintain internal state (e.g., index).
  3. Expose via an Iterable wrapper to support for-each.

Full Example: Iterator for int[]

This example shows a custom iterator IntArrayIterator and a small wrapper collection IntArrayCollection that implements Iterable, enabling the enhanced for-loop.

import java.util.Iterator;
import java.util.NoSuchElementException;

// Custom Iterator for Integer Array
class IntArrayIterator implements Iterator<Integer> {
    private final int[] array;
    private int index = 0;

    public IntArrayIterator(int[] array) {
        this.array = array;
    }

    @Override
    public boolean hasNext() {
        return index < array.length;
    }

    @Override
    public Integer next() {
        if (!hasNext()) {
            throw new NoSuchElementException("No more elements!");
        }
        return array[index++];
    }

    @Override
    public void remove() {
        throw new UnsupportedOperationException("Remove not supported!");
    }
}

// Custom Collection class implementing Iterable
class IntArrayCollection implements Iterable<Integer> {
    private final int[] array;

    public IntArrayCollection(int[] array) {
        this.array = array;
    }

    @Override
    public Iterator<Integer> iterator() {
        return new IntArrayIterator(array);
    }
}

// Demo
public class CustomIteratorExample {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};
        IntArrayCollection collection = new IntArrayCollection(numbers);

        System.out.println("Iterating using custom iterator:");
        for (int num : collection) {
            System.out.println(num);
        }
    }
}

Why this design?

  • Separation of concerns: iterator knows how to traverse; collection exposes iteration.
  • Safe: throws NoSuchElementException if you call next() without a next element.
  • Clear: remove() explicitly unsupported.

Variations You Can Try

  • Filtering: Skip even numbers only iterate odds.
  • Reverse iteration: Start from the end and move backwards.
  • Windowed iteration: Yield chunks (batches) of N elements.

Tip: For filtering, advance the internal index in hasNext() until a matching element is found.
