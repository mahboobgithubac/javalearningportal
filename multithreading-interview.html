Java Multithreading Interview Questions and Answers

Prepare for interviews with essential multithreading concepts, examples, and concurrency tools

1. What is a thread in Java?

A thread is the smallest unit of execution. In Java, threads can run concurrently to perform multitasking.

2. How do you create a thread in Java?

// Extending Thread
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running...");
    }
}

// Implementing Runnable
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable running...");
    }
}

new MyThread().start();
new Thread(new MyRunnable()).start();

3. What is the difference between start() and run()?

  • start() creates a new thread and invokes run().
  • run() executes in the current thread like a normal method.

5. What are daemon threads?

Background threads that exit when all user threads finish.

Thread t = new Thread(() -> System.out.println("Daemon"));
t.setDaemon(true);
t.start();

Used to avoid concurrent access problems on shared resources.

public synchronized void increment() {
    count++;
}

}

Ensures visibility of changes to variables across threads.

volatile boolean flag = true;

11. What is ExecutorService?

Used to manage thread pools and run async tasks.

ExecutorService ex = Executors.newFixedThreadPool(2);
ex.submit(() -> System.out.println("Run"));
ex.shutdown();

12. What is the Callable interface?

Like Runnable but returns a result and can throw exceptions.

Callable<Integer> task = () -> 10;
Future<Integer> future = executor.submit(task);

13. Future vs CompletableFuture

  • Future is blocking
  • CompletableFuture allows chaining and async tasks
CompletableFuture.supplyAsync(() -> "Hello")
    .thenApply(s -> s + " World")
    .thenAccept(System.out::println);

14. Common threading issues

  • Race Condition
  • Deadlock
  • Livelock
  • Starvation

15. What is deadlock and how to prevent it?

  • Avoid nested locks
  • Use lock ordering
  • Use tryLock()

16. ReentrantLock vs synchronized

Allows try-lock, interruptible lock, and fairness control.

Lock lock = new ReentrantLock();
if(lock.tryLock(1, TimeUnit.SECONDS)) {
    try {
        // logic
    } finally {
        lock.unlock();
    }
}

17. Semaphore, CountDownLatch, CyclicBarrier

  • Semaphore: controls access with permits
  • CountDownLatch: waits for tasks to complete
  • CyclicBarrier: waits for threads to reach barrier

18. What is CAS (Compare-And-Swap)?

AtomicInteger atomic = new AtomicInteger(0);
atomic.compareAndSet(0, 1);

19. Fork/Join Framework

class MyTask extends RecursiveTask<Integer> {
    protected Integer compute() {
        // Divide and conquer logic
    }
}

20. What is false sharing?

Occurs when multiple threads access variables on the same cache line, hurting performance. Use padding or separation to avoid.

21. How do you ensure thread safety in Java?

  • Use synchronized blocks or methods
  • Use concurrent collections (e.g., ConcurrentHashMap)
  • Use immutable objects
  • Use atomic variables (like AtomicInteger)

24. What is thread priority?

Thread priority decides the order of thread scheduling. Values range from 1 (MIN) to 10 (MAX), default is 5.

thread.setPriority(Thread.MAX_PRIORITY);

25. What is inter-thread communication in Java?

It allows threads to communicate using wait(), notify(), and notifyAll().

synchronized(obj) {
    while(conditionNotMet) obj.wait();
    obj.notify();
}

26. How do you handle thread interruption?

Use Thread.interrupt() to interrupt a thread and check using Thread.interrupted() or isInterrupted().

while (!Thread.currentThread().isInterrupted()) {
    // work
}

27. Difference between Thread Pool and ForkJoinPool?

  • ThreadPool: Ideal for independent tasks
  • ForkJoinPool: Ideal for recursive, divide-and-conquer tasks

28. What is the use of BlockingQueue?

Used for thread-safe producer-consumer pattern. Blocks automatically when full/empty.

BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);
queue.put("item");
String item = queue.take();

29. How to avoid race conditions?

  • Use synchronized blocks/methods
  • Use atomic classes
  • Minimize shared mutable state

30. Best practices for multithreading in Java

  • Avoid shared mutable state
  • Prefer higher-level concurrency utilities
  • Use thread pools, avoid creating threads manually
  • Always handle exceptions in threads
  • Test for thread-safety using stress tests
