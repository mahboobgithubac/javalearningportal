Dependency Injection and Inversion of Control (IoC) in Java (Spring)

üîÅ What is Inversion of Control (IoC)?

Inversion of Control means reversing the control of object creation from the programmer to a framework (like Spring). The IoC container manages the lifecycle and configuration of application objects.

Note: In traditional programming, your code controls the flow. In IoC, the control is inverted and given to the framework.

üîÑ What is Dependency Injection (DI)?

Dependency Injection is a design pattern and a technique to implement IoC. It means injecting the dependent objects (dependencies) into a class, rather than creating them inside the class.

‚úÖ Types of Dependency Injection in Spring:

  • Constructor Injection
  • Setter Injection
  • Field Injection (via @Autowired)

üí° Real-life Analogy:

Imagine a Car that needs an Engine.

  • Without DI: The Car creates its own Engine.
  • With DI: Someone (a mechanic or container) provides the Engine to the Car.

üìò Example Without Dependency Injection

class Engine {
    void start() {
        System.out.println("Engine started");
    }
}

class Car {
    private Engine engine;

    public Car() {
        engine = new Engine(); // tightly coupled
    }

    void drive() {
        engine.start();
        System.out.println("Car is running");
    }
}

This is tightly coupled and hard to test or replace dependencies.

‚úÖ Example With Dependency Injection (Spring)

1. Model Classes

// Dependency
@Component
class Engine {
    public void start() {
        System.out.println("Engine started");
    }
}

// Dependent
@Component
class Car {
    private Engine engine;

    // Constructor Injection
    @Autowired
    public Car(Engine engine) {
        this.engine = engine;
    }

    public void drive() {
        engine.start();
        System.out.println("Car is running");
    }
}

2. Configuration and Main Class

@SpringBootApplication
public class App {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(App.class, args);
        Car car = context.getBean(Car.class);
        car.drive();
    }
}
