E-Commerce System Design

Design involves product catalog, cart, order management, inventory, payment, search, and scalability.

  • Services: User, Product, Order, Payment
  • Database: SQL + Redis cache
  • API Gateway + Load Balancer
  • Search: Elasticsearch
  • Queue: RabbitMQ for order/payment events

Chat Service Design

Real-time communication using WebSockets, message queues, and persistent storage.

  • Protocol: WebSocket (or HTTP long-polling)
  • Queue: Kafka for storing messages
  • Database: MongoDB or Cassandra
  • Delivery: Push notifications for offline users

URL Shortener Design

Maps long URLs to short keys with expiration and redirection logic.

  • Hashing or Base62 encoding
  • Cache layer for fast redirection
  • DB: Key-Value Store (Redis, DynamoDB)
  • Analytics and logging

Notification System Design

Send push/email/SMS notifications with retries and logging.

  • Producer: Microservices publish events
  • Queue: Kafka/SQS
  • Consumers: Email/SMS/Push services
  • Logging and retry mechanism

Video Streaming Platform

Stores, encodes, and streams video content efficiently.

  • Upload and transcoding service
  • Storage: S3/GCS
  • CDN for video delivery
  • Database for metadata

Load Balancer

Distributes incoming traffic across multiple backend servers.

  • Algorithms: Round Robin, Least Connections
  • Health Checks
  • Failover and redundancy

Content Delivery Network (CDN)

Delivers static content like images, CSS, JS from edge servers close to users.

  • Reduce latency
  • Cache invalidation rules
  • Examples: Cloudflare, Akamai

Rate Limiting

Controls the number of requests per user/system to avoid abuse.

  • Algorithms: Token Bucket, Leaky Bucket
  • Implementation: Redis + counters
  • Client identification via IP or token
