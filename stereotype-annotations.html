üî∞ Stereotype Annotations in Spring Boot

Stereotype annotations are special annotations that mark a class as a Spring-managed component. These annotations help Spring detect and manage different layers like controller, service, and repository using component scanning.

1. @Component

Marks a class as a generic Spring-managed component. It's the base annotation for others like @Service, @Repository, and @Controller.

@Component
public class MyComponent {
    public void doWork() {
        System.out.println("Working...");
    }
}

2. @Service

Used to mark business logic layer classes. Also supports AOP.

@Service
public class UserService {
    public void processUser() {
        System.out.println("Processing user...");
    }
}

3. @Repository

Marks a class as a Data Access Layer component. Provides automatic exception translation.

@Repository
public class UserRepository {
    public void save() {
        System.out.println("Saving user...");
    }
}

4. @Controller

Marks a class as a web controller in Spring MVC.

@Controller
public class UserController {
    @GetMapping("/hello")
    public String hello() {
        return "hello.jsp";
    }
}

5. @RestController

Combination of @Controller and @ResponseBody. Used for REST APIs to return JSON/XML.

@RestController
public class ApiController {
    @GetMapping("/api/data")
    public String getData() {
        return "Some data";
    }
}

üß† How Stereotype Annotations Work

When a class is annotated, Spring:

  • Detects it via @ComponentScan (enabled via @SpringBootApplication)
  • Manages it as a bean in the ApplicationContext
  • Allows it to be injected using @Autowired or constructor injection

üîç Component Scanning Example

@SpringBootApplication
public class MyApp {
    public static void main(String[] args) {
        SpringApplication.run(MyApp.class, args);
    }
}

All stereotype-annotated classes in the same or sub-packages of MyApp will be scanned.
