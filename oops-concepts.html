Java is a pure object-oriented language (with few exceptions like int, char, etc.) that supports core OOP principles to design modular and reusable software.

A blueprint for creating objects. Contains fields (variables) and methods.

    }
}

ðŸ”‘ 2. Object

A real-world entity created from a class. Has state (fields) and behavior (methods).

ðŸ”‘ 3. Encapsulation

Wrapping data (variables) and methods in a class. Achieved using private fields and public getters/setters.

class Account {
    private int balance;

    public int getBalance() { return balance; }
    public void setBalance(int amount) { balance = amount; }
}

ðŸ”‘ 4. Inheritance

One class inherits the properties and behaviors of another. Promotes code reuse.

class Animal {
    void sound() {
        System.out.println("Animal sound");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks");
    }
}

ðŸ”‘ 6. Abstraction

Hiding implementation details and showing only essential features. Achieved using abstract classes or interfaces.

abstract class Shape {
    abstract void draw();
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing Circle");
    }
}

ðŸ”‘ 7. Association, Aggregation, Composition

  • Association: General relationship (e.g., teacher and student)
  • Aggregation: Has-a, weak ownership (e.g., department has professors)
  • Composition: Strong ownership (e.g., house has rooms)
