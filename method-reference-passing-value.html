Passing Values to Method References (Lambda vs Method Reference)

Short guide showing how values are passed when using method references and when you must use lambdas instead.

How values get passed

When you use a method reference, the values are supplied by the calling context (the functional interface method). You don't manually pass parameters — the functional interface's abstract method provides them.

1. Example: forEach with System.out::println

// Lambda
names.forEach(name -> System.out.println(name));

// Method reference (equivalent)
names.forEach(System.out::println);

Here, forEach provides each element to System.out.println automatically.

2. Static method reference

// MyUtils::getLength example
import java.util.function.Function;

Function strToLen = MyUtils::getLength;

System.out.println(strToLen.apply("MethodRef")); // value passed by apply()

class MyUtils {
    public static int getLength(String s) { return s.length(); }
}

3. Instance method reference (bound to object)

// Bound instance method reference
MyMath math = new MyMath();
BiFunction add = math::add;
System.out.println(add.apply(10, 4)); // 10 and 4 passed by apply()

class MyMath {
    public int add(int x, int y) { return x + y; }
}

4. When you need extra arguments

If your target method requires more arguments than the functional interface supplies, you cannot use a plain method reference; use a lambda to provide the extra (fixed) values.

// Suppose printWithPrefix(String prefix, String msg)
// Consumer<String> gives only one String (msg)
// So you must use a lambda:
Consumer c = msg -> MyUtils.printWithPrefix("INFO: ", msg);

// You cannot write: MyUtils::printWithPrefix here, because the signatures don't match.

Rule of thumb

  • Method reference works when the functional interface method's parameters exactly match the target method's parameters (or match via instance binding).
  • If you need to supply extra fixed arguments or perform additional logic, use a lambda instead.
  • Method references are syntactic sugar — prefer them when they make code clearer and concise.
