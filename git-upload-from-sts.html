🚀 How to Push Code to GitHub from STS (Spring Tool Suite)

1. Initialize Git in Your Project

Right-click your project → Team → Share Project → Select Git and click Finish.

2. Commit Your Code

Right-click your project → Team → Commit.

  • Select files to commit
  • Add a commit message
  • Click Commit or Commit and Push

3. Create a GitHub Repository

Go to GitHub → Click New Repository → Fill in details → Click Create Repository.

4. Connect to Remote Repository

Right-click project → Team → Remote → Push → Add a new remote:

  • Remote name: origin
  • Remote URI: https://github.com/your-username/your-repo.git
  • Provide GitHub credentials if prompted
  • Click Next → Select branch (e.g., master or main) → Finish
🔒 Push Disabled?
You must first commit at least one change before push becomes available.

5. Push Changes

After committing, right-click project → Team → Push to Upstream.

You can now see your project on GitHub!

🔁 How to Push Future Updates

  1. Make your code changes
  2. Right-click project → Team → Commit
  3. Add a commit message → Click Commit and Push

This will update the remote GitHub repository with your latest changes.

📦 Tip: Check Status

You can always use the Git Staging View in STS to see which files are staged, modified, or untracked.

💡 Pro Tip:
Make sure your GitHub repository is private or public as per your preference. Don't forget to push regularly!
