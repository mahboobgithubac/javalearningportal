Intermediate operations return a new stream. They are lazy and not executed until a terminal operation is invoked.

  • map(): transforms each element
  • filter(): filters elements by predicate
  • sorted(): sorts elements
  • distinct(): removes duplicates
  • limit(), skip(): truncates the stream

import java.util.Arrays;
import java.util.List;
import java.util.stream.Stream;

public class IntermediateExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Tom", "Jerry", "Spike", "Tom");

        names.stream()
             .filter(name -> name.length() > 3)
             .map(String::toUpperCase)
             .distinct()
             .sorted()
             .forEach(System.out::println); // Terminal Operation
    }
}
Output:
JERRY
SPIKE

ğŸ“Œ What Are Terminal Operations?

Terminal operations produce a result or side-effect and trigger the actual processing of the stream.

  • forEach(): performs an action for each element
  • collect(): reduces elements to a collection
  • count(): returns number of elements
  • reduce(): combines elements into one
  • anyMatch(), allMatch(), noneMatch(): test conditions

import java.util.Arrays;
import java.util.List;

public class TerminalExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

        long count = numbers.stream()
                            .filter(n -> n % 2 == 0) // Intermediate
                            .count();               // Terminal

        System.out.println("Even count: " + count);
    }
}
Output:
Even count: 2
