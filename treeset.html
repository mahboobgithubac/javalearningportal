Understand TreeSet with Examples and Best Practices

‚úÖ TreeSet in Java (with Example)

A TreeSet in Java is part of the java.util package and implements the Set interface. It uses a Red-Black Tree internally to store elements in sorted order and does not allow duplicates.

üîç Key Features of TreeSet:

  • Sorted: Elements are automatically stored in ascending order (natural ordering) or by a custom comparator.
  • No Duplicates: Duplicate elements are ignored.
  • Null: Allows at most one null element, and only if using natural ordering.
  • Thread-safe: Not synchronized; use Collections.synchronizedSet() for thread safety.

üì¶ TreeSet Declaration:

TreeSet<Type> setName = new TreeSet<>();

‚úÖ Example: Natural Ordering (Ascending)

import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        TreeSet<Integer> numbers = new TreeSet<>();

        numbers.add(40);
        numbers.add(10);
        numbers.add(30);
        numbers.add(20);
        numbers.add(10); // duplicate, will be ignored

        System.out.println("TreeSet (Sorted): " + numbers);
    }
}

Output:
TreeSet (Sorted): [10, 20, 30, 40]

‚úÖ Example: String Sorting

TreeSet<String> names = new TreeSet<>();
names.add("Alice");
names.add("Charlie");
names.add("Bob");

System.out.println(names); // Output: [Alice, Bob, Charlie]

‚úÖ Example: Custom Comparator (Descending Order)

import java.util.*;

public class CustomComparatorTreeSet {
    public static void main(String[] args) {
        TreeSet<Integer> set = new TreeSet<>(Collections.reverseOrder());

        set.add(100);
        set.add(50);
        set.add(75);
        set.add(25);

        System.out.println("TreeSet Descending: " + set);
    }
}

Output:
TreeSet Descending: [100, 75, 50, 25]

‚ö†Ô∏è Common Pitfalls:

  • Adding incompatible types (e.g., integers and strings) will throw ClassCastException.
  • NullPointerException if null is added in a non-empty set or with custom comparators.

üìö When to Use TreeSet?

  • When you need sorted elements.
  • Duplicates are not allowed.
  • Fast search, removal, and insertion in sorted order.

Tip: Use TreeSet for sorted high scores, usernames, or anytime ordering matters!
