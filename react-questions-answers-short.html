React & Redux Interview Questions

React is an open-source JS library by Facebook for building UI components in single-page applications. Uses component-based, declarative approach with virtual DOM.

  • JSX – Syntax to define UI
  • Components – Reusable building blocks
  • Virtual DOM – Efficient updates
  • One-way data binding – Predictable flow
  • Declarative UI – Describe how UI looks
  • Hooks – Functional state & lifecycle

JSX stands for JavaScript XML. It allows writing HTML-like syntax in JS, which gets transformed to React.createElement calls.

Virtual DOM is an in-memory representation of the real DOM. React diff compares new virtual DOM to previous one, then applies minimal updates to real DOM.

Functional components are plain JS functions that can use hooks. Class components use ES6 classes and lifecycle methods (render, componentDidMount, etc.).

Props are inputs passed to components. They are read-only and allow parent components to configure children.

State is internal data managed within a component, usually via useState (functional) or this.state (class). Changing state triggers re-renders.

React uses synthetic events to normalize browser differences. Events are passed as camelCase props (e.g., onClick).

Key helps React identify which items have changed, added, or removed. Should be unique and stable across renders.

Controlled components have inputs managed by React state. Uncontrolled use refs and DOM for values.

useState adds state to functional components. Returns a state variable and setter function.

useEffect runs side effects after render. You can specify dependencies, cleanup functions, and run-once or run-every-update effects.

The dependency array controls when effect runs. Empty means run once on mount. Missing array runs on every render. Listing values triggers on their change.

Hooks let functional components use state and lifecycle. Common: useState, useEffect, useContext, useRef, useCallback, useMemo.

useRef holds a mutable object that persists across renders without causing re-render. useState updates cause renders.

Context provides a way to pass data through component tree without passing props manually. Ideal for themes, auth, locale.

Prop drilling is passing props through intermediate components. Avoid via context or Redux.

  • Memoize components (React.memo), values (useMemo), and callbacks (useCallback)
  • Lazy load components (React.lazy)
  • Avoid unnecessary re-renders
  • Use virtualization for lists

Use JS conditions: ternary (cond ? : ), && operator, or conditional components.

Memoization caches values or components to avoid recalculations/rerenders. React.memo wraps components; useMemo caches values between renders.

Reconciliation is React’s diffing process. It compares virtual DOM trees and updates only changed parts in the real DOM.

An HOC is a function that takes a component and returns a new component with added functionality.

Render props pattern passes a function prop to component which returns JSX, allowing dynamic rendering based on logic shared.

useCallback memoizes a function, preventing re-creation unless dependencies change; useful for passing stable callbacks.

Lazy loading delays loading components until needed. Use React.lazy and Suspense:

const MyComp = React.lazy(() => import('./MyComp'));

Error boundaries are class components implementing componentDidCatch to catch errors in subtree and render fallback UI.

forwardRef passes ref through component to a child DOM or class component: React.forwardRef.

Suspense lets UI “wait” for async data/components. Concurrent rendering allows React to interrupt renders to keep UI responsive.

SSR renders React components to HTML on server and sends it to client for faster load and SEO.

Hydration is attaching event listeners to SSR-rendered HTML on client to make it interactive.

Redux is a predictable state container for JS apps with a single source of truth and strict unidirectional data flow.

  • Single source of truth
  • State is read-only
  • Changes via pure functions (reducers)

Store holds the complete app state and provides methods like getState(), dispatch(), and subscribe().

Action is a plain JS object describing state change, with type and optional payload.

A reducer is a pure function (state, action) → new state. No side-effects.

dispatch() sends an action to the store, triggering reducers and state updates.

connect() links React components to Redux store, mapping state and dispatch to props.

Middleware intercepts actions between dispatch and reducer to enable async, logging, error handling, etc.

Define actions/reducers, create store, wrap app with <Provider>, dispatch actions from components.

Context is for passing data through tree; Redux is a full state-management solution with reducers, middleware, devtools.

Redux-thunk and Redux-saga are middleware for handling async operations. Thunk uses functions, Saga uses generator effects.

useSelector selects state from Redux store; useDispatch returns dispatch function to send actions.

<Provider> from react-redux wraps your app and makes the Redux store available to nested components.

Use async middleware like thunk or saga, dispatch request/success/failure actions around async calls.

RTK simplifies setup with createSlice, createAsyncThunk, built-in devtools. Downsides: magic abstracts Redux, learning curve.

Slices group reducers, actions, and initial state for a feature using createSlice().

State is not mutated directly. Reducers return new state. Libraries like Immer simplify immutable updates.

Use Redux DevTools, add logging middleware, inspect state and actions in console/network.

Use localStorage or sessionStorage plus libraries like redux-persist to rehydrate store on load.

RTK reduces boilerplate with createSlice, configureStore vs Redux’s manual action types, creators, combineReducers, applyMiddleware.
