1. Array

Definition: A collection of elements stored in contiguous memory locations.

Example: int[] arr = {1, 2, 3, 4};

Use Case: Random access, static collections.

2. Linked List

Definition: A linear collection of elements, where each element points to the next.

Types: Singly Linked List, Doubly Linked List, Circular Linked List

Use Case: Efficient insertions/deletions.

3. Stack (LIFO)

Definition: Last-In, First-Out structure.

Operations: push(), pop(), peek()

Use Case: Undo features, function call stack, expression evaluation.

4. Queue (FIFO)

Definition: First-In, First-Out structure.

Types: Simple Queue, Circular Queue, Priority Queue, Deque (Double-Ended Queue)

Use Case: Scheduling tasks, buffer management.

5. HashMap / HashTable

Definition: Key-value pair storage using hashing.

Use Case: Fast lookup, indexing, caching.

6. Set

Definition: A collection of unique elements.

Types: HashSet, TreeSet

Use Case: Duplicate elimination, fast membership tests.

7. Tree

Definition: Hierarchical data structure with a root node and child nodes.

Types: Binary Tree, Binary Search Tree (BST), AVL Tree, B-Tree, B+ Tree, Heap

Use Case: Searching, sorting, hierarchy modeling.

8. Graph

Definition: A collection of nodes (vertices) connected by edges.

Types: Directed/Undirected, Weighted/Unweighted, Cyclic/Acyclic

Use Case: Network routing, social networks, pathfinding.

9. Trie (Prefix Tree)

Definition: A tree used for storing strings, where each node represents a character.

Use Case: Autocomplete, spell checking, dictionary lookup.

10. Heap

Definition: A complete binary tree used to implement priority queues.

Types: Min-Heap (root is minimum), Max-Heap (root is maximum)

Use Case: Priority queue, heap sort, scheduling.
